Element Attributes

Elements often have multiple attributes associated with them.

Link to Google

Attributes are written key="value" like in href="www.google.com" or src="page.html". They're separated by spaces and their value

Anchor

Anchors (links) take the user to another page. *usually

The attribute specifies the target URL of the link. Uniform Resource Locator strings can, among other things, access other websites, other html files, and specific places on your website.

URLs

facebook.com/Google/ is an absolute URL. You could copy it into your browser's search bar and you would end up on Google's Facebook page. An absoute url, like this one, leaves no question about where it's pointing to. It has the domain facebook.com which points to Facebook's servers, and then the path /Google/ which says exactly what page on their servers it wants (Google's Facebook page in this case).

Targets

Anchors also have the target attribute. This specifies what the browser will do to the link.

_blank Opens the link in a new tab.
_self Opens the link in current view.
_parent Opens the link in the parent frame.
_top Replaces the current tab's content.
framename Opens the linked document in the iframe with the attribute name=framename

src vs href

There is a rational behind making the src and href attributes have different names.

href is a shortening of Hyperlink Refrence, and unlike src, the resource refrenced doesn't need to be located before the page can load.

Frames

Frames contain their own html. They allow modularity in pages. This lesson is in a frame, and that frames content is changed when you switch lessons. This lets me keep all the lessons in separate html files.

The iframe element displays the url given by the src attribute.

There's some controversy online about if iframes should be used or not. I haven't dug far into it, but it looks like is the current way to do this, and is depreciated.
