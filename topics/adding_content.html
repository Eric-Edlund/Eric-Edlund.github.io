URLs

Uniform Resource Locator strings are the street addresses of computers. They can point to websites, files, and even specific places on your website.

facebook.com/Google/ is an absolute URL. You could copy it into your browser's search bar and you would end up on Google's Facebook page. An absoute url leaves no question about where it's pointing to because it has no context. It has the domain facebook.com which points to Facebook's servers, and the path /Google/ which says exactly what page on their servers it wants (Google's Facebook page in this case).

Relative URLs lack an explicit domain, leaving it up to context. Usually, the path given is interpreted to be relative to the current file directory of the html file on the host computer. href vs src

href and src attributes both take a URL and are present in multiple elements. They have a slight semantic difference. src is used for substitution. In the img element, the content of the image is specified with src because the image will fill the content of the img. The browser stops for every src element to try and fetch the content before going on reading the document.

href specifies a refrence to an external resource. Parsing of the page isn't stopped while we fetch the resource. a (anchor element) uses href because the content stored at the link doesn't replace anything in the page.

id

The id attribute can be specified on any element. As the name suggests, its value is a identifier, unique among other ids specified on the page. This is useful for refrencing objects later.

A URL pointing to a webpage can have a fragment suffix which refrences a specific element within the webpage by its id. In the navigation section of the left of this page, the subheadings bring you to a specific section in a topic.

Special Value Attributes

Many elements have unique attributes with a set of allowed values. An example of this is the target attribute in the a element, which determines where the link will be opened. Here are all possible values:

_blank Opens the link in a new tab.
_self Opens the link in current view.
_parent Opens the link in the parent frame.
_top Replaces the current tab's content.
framename Opens the linked document in the iframe element with the attribute name=framename

Frames

Frames contain their own HTML, allowing for modularity in pages. I have put this lesson in an iframe to the right. The rectangle containing the normal sized lesson is also in an iframe. When you click a topic in the navigation, the content of this iframe is replaced with the lesson you asked for. This lets me keep all the lessons organized in separate HTML files.

The iframe element displays the url given by the src attribute.
