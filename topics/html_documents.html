The content of a webpage is stored in Hyper Text Markup Language (HTML). HTML code is stored in plain text files ending in .html.

Elements

An HTML document is a collection of Elements, each notated with an opening tag, like , and a matching closing tag, like . An element is really just a thing.

There are many different types of elements (things) that you can have on your website (currently, this html document). What type of element you have is specified in the opening and closing tags. A , for example, could be a cat element, while could represent a dog.

Something very important to notice is that does not constitute an element. Upon arriving a line like this, a web browser (the thing drawing your website!) would have no idea what you were saying. Closing tags need to match opening ones.

Sadly, the element isn't a part of the HTML standard, and so a web browser wouldn't know what to do with it if you used one.

For an html file to be a proper website, there are a few important elements it needs to have. Firstly, needs to appear at the top of the document. This tells the browser what it's looking at. Under that, you need an html element. In HTML, spaces, indentation and new lines don't affect how the document will be read, so you should use those things to produce an organized, human readable document. As long as the html element appears after that heading, it should work.

Nested Elements

The html element will contain all the information about the web page. Inside the html element, between it's opening and closing tags, you may put other nested elements. Elements inside other elements take on a parent-child relationship. The elements inside an element are that element's children, and the element surrounding a child element is its parent. An element can have multiple children (who are siblings to each other).

The html element of a website has two important children:

  1. The element contains data about the website, like what language it's in and what should be displayed on the tab. Nothing in the head will be displayed on the page.
  2. The contains the content on the page. Its children hold text, images, links, videos and whatever else your site has on it.

In the body, there is only 1 element. The first thing, , is a comment. It's contents will be ignored by the browser and its only there for humans to make notes. The element that the browser will see is the paragraph element, notated as just p. Paragraphs will format their text to make it look like a parargaph. p is, again, one of many many different elements in html you can use. There are far too many to cover, but I'll go over some of them in the next topic.

Helpful Tools

An IDE like Visual Studio Code is highly recommended as it will bring small, non-obvious yet frustrating issues in your html code to your attention. It will prevent you from spending an hour searching over your code trying to figure out why it won't work and then realizing it was because you forgot to put a ", or close an element, or some other insignificant detail you shouldn't spend time thinking about. Frequently, it will also be able to explain your mistake and offer a solution to fix it. Your IDE will be like a set of training wheels for now, and provide other powerful tools to you later.

There are a hundred IDEs out there, and just like spirit animals and wands, you have to find yours.
