Stylizing Elements

CSS is a language that's used with HTML to adjust how elements look. If HTML was all the nouns in a webpage, CSS would be all the adjectives.

p, body, and h1 are all element selectors. By using just the tag name of any html element, all elements of that type will be selected. Multiple tags separated by spaces, like p h1, will select the union of those selectors (all paragraphs and all level 1 headers).

Attribute Selectors

You can also discriminate elements by their attributes.

If you specify the id attribute of an element () you can then refrence that element with the selector #paragraph1. This selector will only apply the rule to the first element it finds with the given id. Elements should have unique ids.

The class attribute doesn't need to be unique. It should be used to identify multiple elements that all require common formatting. The selector for class type is .className.

Class names and IDs can't contain spaces. An element can have multiple classes specified by putting spaces in the class attribute like class="c1 c2".

You can specify by any arbitrary attribute with the attribute selector [attribute="value"]. There are a lot of little tricks you can do with this w3Schools .

Attribute selectors can be prefixed by an element selector. p.special will select only paragraphs with the special class. a[target="_blank"] only links that will open their content in a new page. Attribute selectors can also be stacked, refrencing more and more specific groups. a.glowing[target="_self"] selects links with the glowing class that will replace the current page when clicked.

Declarations

One rule will usually contian multiple selectors. Again, they take the form attribute-name: value;, as in font-size: large;. CSS attributes don't use camel case (capitalizingTheFirstLetterOfWords), instead they separate words with -, as in text-align. CSS naming convention is a big thing you can read about.

There are quite a few attributes. I'll list a few here, but generally new attributes are discovered by googling How to do ____ in css.

The colors of an element can be set with color and background-color. CSS has quite a few standard color names like orange, tomato, coral, darkgreen, and white. You can also specify rgb values with rgb(red, green, blue) where each color is an integer [0, 255]. You can also use rgba(...) and specify a final alpha value for transparency [0 (transparent), 1 (opaque)]. Hex values can be prefixed by # as like in #FFFFFF.

The background of an element can be set using any of the background- properties discussed here. This paragraph has been formated with background-color: lightgray.

You can add borders to an element with the border- elements. There are also many convenient shorthand attributes. border can be used to specify multiple details about a border at once. This paragraph has been formatted with border: black dotted 3px;

With the more specific border- attributes, you can specify border-width, border-color, border-style (dotted, dashed, solid...), border-radius (for rounded borders), and attributes specific to each border separately: border-top-color, border-left-style.

CSS Types

Different attributes demand different types of values. There isn't a compile-check for types as seen in languages like Java. Instead, if you specify a value that doesn't make sense, CSS will just ignore it and continue on.

All css attributes will take initial, inherit, or unset, which define the value relative to it's default value or the parent element.

Attributes like border-style take pre-defined keywords, in it's case, dotted | dashed | solid | groove | ridge | inset | outset | none | hidden.

background-image requires a URL value, notated url("path/to/image").

Cascading Style

CSS Stands for Cascading Style Sheet. This name points out a central property of css: that later declarations override earlier ones.

In the declaration list for a rule, if color: orange; succeeds color: white;, color will be set to orange. A succeeding rule with the same selector will only override the values of specified attributes; p { color: gray; background-color: green } followed by p { color: red } will have red text on a green background.

Similarly, when CSS is put in multiple places, the last CSS that is read takes precident. If you link a style sheet that makes green, then have a element later in your HTML that specifies white, the header will be white because the style element was read after the linked stylesheet.

The body element will end with the value white as it's background-color
